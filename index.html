はじめに

ここでは、Minecraft for Windowsにおける改造クライアントの開発の基礎を解説します。コードの書き方は、概ねIrisクライアントに改善を加えたものを参考にします。以下の前提条件を確認し、次のステップに進んでください。

前提

✅ Visual Studio 2026をインストールしている

✅ Minecraft for Windows（バージョン26.10以上）をインストールしている

※以降の解説はあくまでも基礎であり、安全かつ最適であるとは言えません。

プロジェクトの作成

それでは、プロジェクトを作成しましょう。Visual Studioを開き、右上にある「Create a new project」をクリックします。日本語で使用している人は、次の写真を参考にしてください。

次に、プロジェクトの種類「Dynamic-Link Library (DLL)」を選択し、ダブルクリックしてください。

プロジェクトを作成します。プロジェクトの名前とファイルパスを指定し、右下の「Create」をクリックします。

（この後のプロパティ設定についての説明は端折ります。）

dllmain.cppのセットアップ

dllmain.cppは、アドレス（シグネチャやオフセット）や機能（フックやモジュール）の初期化・解放など、クライアントの基本処理のすべてを司る重要なファイルの1つです。初期状態では、pch.hのインクルードに加え、DllMain関数が用意されています。

アドレスや機能の初期化は、初期化用のスレッド関数にて行います。ここではDWORD InitializeClient(HMODULE hModule)を初期化スレッドとして定義します。

また、Irisクライアントではプリコンパイル済みヘッダ（pch）を使用しないため、pch.hの代わりに、まずWindows.hをインクルードしてください。以上を踏まえたdllmain.cppのコード例を以下に提示します。


            #include <windows.h>

            DWORD InitializeClient(HMODULE hModule) {
              return 0;
            }

            BOOL APIENTRY DllMain(HMODULE hModule, DWORD ul_reason_for_call, LPVOID lpReserved) {
              if (ul_reason_for_call == DLL_PROCESS_ATTACH) {
                DisableThreadLibraryCalls(hModule);
                CreateThread(0, 0, (LPTHREAD_START_ROUTINE)InitializeClient, hModule, 0, 0);
              }
              return TRUE;
            }

ゲーム関数のフック

ここからは、ゲーム内の関数をフックする方法を解説します。ここではsafetyhookとlibhatを使用します。また、今回はKeymapを例にフックします。

Keymapは、関数void Keyboard::_feed(Keyboard* this, __int32 key, bool held)のことを指し、ホットキーの指定に使用されます。

まずはKeyboard::_feedのシグネチャを取得します。取得方法はYouTubeに投稿されているものなどを参考にしてください。（https://www.youtube.com/watch?v=jIZnWKZwjVc）

26.10現在でのシグネチャは4C 8B DC 48 81 EC ? ? ? ? 81 FAです。これをアドレスとしてフックします。

まず、このシグネチャをスキャンする関数を用意します。dllmain.cppとは別のファイルに用意することを推奨します。以下に、libhatを用いた例を提示します。


            static hat::scan_result scanSig(std::string_view sig) {
            	auto parsed_sig = hat::parse_signature(sig);
            	const auto result = hat::find_pattern(parsed_sig.value(), ".text");
            	return result;
            }

次に、フックを管理するファイルを作成します。以下にコード例を提示します。


            #pragma once

            class FuncHook {
            public:
              virtual void Initialize() = 0;
            }

            #include "Hooks/Keymap.h"

            void InitializeHooks() {
              static FuncHook* hooks[] = {
                &KeymapHook::Instance()
              };

              for (std::size_t i = 0; i < std::size(hooks); ++i) {
                hooks[i]->Initialize();
              }
            }

この時点ではKeymap.hとその中身を作成していないためエラーが出ていますが、後に解消されます。

次に、Keymapをフックするファイルを作成します。フックを管理するファイルと同じディレクトリにHooksというフォルダを作成し、その中にフックするファイルを作成することを推奨します。以下にコード例を提示します。


            #pragma once

            safetyhook::InlineHook oKey;

            void KeymapDetour(void* _this, __int32 key, bool held) {
              auto& hook = oKey;
              hook.call<void>(_this, key, held);
            }

            class KeymapHook : public FuncHook {
            public:
              void Initialize() {
                uintptr_t sig = scanSig("4C 8B DC 48 81 EC ? ? ? ? 81 FA");
                oKey = safetyhook::create_inline(sig, (void*)&KeymapDetour);
              }

              static KeymapHook& Instance() {
                static KeymapHook instance;
                return instance;
              }
            }

フックの仕組みは完成したので、dllmain.cppのDWORD InitializeClient(HMODULE hModule)関数内にフックを初期化する処理を加えます。以下にコード例を提示します。


            DWORD InitializeClient(HMODULE hModule) {
              InitializeHooks();

              return 0;
            }

以上で、Keymapのフックが完成されました。

ClientInstanceの取得

ClientInstanceは、getLocalPlayer()やgetMinecraftGame()など、ゲームを改造する上で必須となるような関数を多く持っています。

今回は、ScreenView::setupAndRender(ScreenView*, MinecraftUIRenderContext*)という関数をフックして取得します。（フックの手順はKeymapと殆ど同じなので省略します。）

まず、新たにSDK（Software Development Kit = ソフトウェア開発キット）用のフォルダを作成し、その中にClientInstance.hとMinecraftUIRenderContext.hを作成します。

次に、ClientInstance.hの中身を記述します。今は一旦、ClientInstanceクラスのみを定義します。以下のように記述してください。


            #pragma once

            class ClientInstance {
            public:
            };

MinecraftUIRenderContext.hの中身を記述します。まず、MinecraftUIRenderContextクラスを定義します。 次に、MinecraftUIRenderContext::mClientという変数のオフセットを取得します。 取得方法は省略しますが、26.10現在までオフセットは0x8で固定されています。（※勿論、今後のアップデートで変更される可能性があります。）

取得したオフセットにアクセスし、ClientInstanceを読むコードを記述します。今回はマクロを使用します。以下のようなマクロを定義してください。


            #define AS_FIELD(type, name, fn) __declspec(property(get = fn, put = set##name)) type name

            #define BUILD_ACCESS(ptr, type, name, offset) \
            AS_FIELD(type, name, get##name); \
            type get##name() const { return *reinterpret_cast((uintptr_t)ptr + offset); } \
            void set##name(type v) const { *reinterpret_cast((uintptr_t)ptr + offset) = std::move(v); }

最終的に、MinecraftUIRenderContext.hの中身は以下のようになります。


            #pragma once

            class MinecraftUIRenderContext {
            public:
              BUILD_ACCESS(this, ClientInstance*, mClient, 0x8);
            };

次に、mClientを格納する用のポインタを用意します。 今回は、Globals.hというファイルを作成し、その中に定義します。 以下のように記述してください。


            #pragma once

            class Globals {
            public:
              static class ClientInstance* ClientInstance;
            };

ScreenView::setupAndRender関数をフックするファイルに、Detour関数を次のように定義します。


            safetyhook::InlineHook oRender;

            void RenderDetour(void* _this, MinecraftUIRenderContext* ctx) {
              Globals::ClientInstance = ctx->mClient;

              auto& hook = oRender;
              hook.call(_this, ctx);
            }

最後にこのDetour関数とScreenView::setupAndRender関数のシグネチャをもってフックします。例によってシグネチャの取得方法とフック方法は省略します。

26.10現在のシグネチャは48 89 5C 24 ? 55 56 57 41 54 41 55 41 56 41 57 48 8D AC 24 ? ? ? ? 48 81 EC ? ? ? ? 0F 29 BC 24 ? ? ? ? 48 8B 05 ? ? ? ? 48 33 C4 48 89 85 ? ? ? ? 4C 8B FAです。

これで、ClientInstanceを取得することができました。使用例は以下のようになります。


            Globals::ClientInstance->getLocalPlayer();
            Globals::ClientInstance->getMinecraftGame();

変数・関数

変数とは、数値や文字列などの値を記憶・変更するための「名前付きのコンテナ」のことです。以下が変数の例です。


            bool boolean = true;
            int integer = 50;
            float fraction = 20.f;
            std::string text = "abc";
            std::vector<int> intVec;

関数とは、特定の処理をまとめて、必要な時に再利用できるコードの「名前付きのブロック」のことです。以下が関数の例です。


            void funcA() {
              std::cout << "This is a message from funcA" << std::endl;
            }

            bool funcB() {
              return true;
            }

            float funcC() {
              return 5.f;
            }

関数は、引数と呼ばれる値を持つことができます。引数とは、関数を呼び出す際に、その関数に処理に必要なデータを受け渡すための変数や値のことを指します。例としては、 void funcD(const std::string& str, bool state, int index)などです。

この丸括弧の中にカンマで区切られて並んだ引数の集合を引数リストと呼びます。

データ型

C++には、データ型と呼ばれるものが存在します。これは、変数の種類を定めるのに使うもので、代表的な物は、bool、int、floatなどがあります。

bool = true/falseで表す。

int = 整数。

float = 小数。

宣言・定義

関数や変数の存在をコンパイラに紹介することを、関数や変数を宣言するといいます。以下が例です。


            int mode; // 変数 "mode" を宣言
            void func(int index); // 関数 "func" を宣言

宣言した関数や変数の内容を明確にすることを、関数や変数を定義するといいます。以下が例です。


            int Class::mode = 0; // 構造体 "Class" で宣言した変数 "mode" を具体化。

            void Class::func(int index) {
              if (index < 10) {
                std::cout << "current index is smaller than 10" << std::endl;
              }
            } // 構造体 "Class" で宣言した関数 "func" を具体化。

※宣言を行わず、定義のみを行うケースもあります。

構造体

「データ」と「処理」をまとめたものを構造体と呼び、これにはclassとstructの2種類があります。

構造体は3つのアクセス修飾子を持ちます。アクセス修飾子の役割は、クラスのメンバ（変数や関数）にどこからであればアクセスできるかを指定することです。これには、public:、private:、protected:があります。

public:は、クラス内、継承先の子クラス内、クラス外の全てからアクセスできます。

private:は、クラス内からのみアクセスできます。

protected:は、クラス内と継承先の子クラス内からアクセスできます。

実際のアクセス修飾子を指定するには、以下のようにコードを記述します。


            class Parent {
            public:
              int publicVar = 0; // 親クラス、子クラス、クラス外
            private:
              int privateVar = 0; // 親クラス
            protected:
              int protectedVar = 0; // 親クラス、子クラス
            }

ただし上記のコードはclassの場合であり、structの場合は元がpublic:の状態になっています。

定数

確定され、変化することのない値を定数といいます。定数を定義するにあたって、constとconstexprという2つの型修飾子を使うことができます。

2つの型修飾子の大きな違いは、値が確定するタイミングにあります。 constは、プログラムを実行している最中に値が確定します。 一方でconstexprは、コンパイル時に値が確定するので、変数を定義した時点で値を1つに確定している必要があります。

以下が例です。


            // const -> コンパイル時に値が確定しないため、ランダムな値を持つことができる。
            const int var1 = rand();

            // constexpr -> コンパイル時に値が確定するため、一つの値しか持つことができない。
            constexpr int var2 = 500;
上で記述したように、constexprはコンパイル時に値が確定するため、必然的に計算の回数が減り、パフォーマンスの向上に繋がります。

if文・else文

「Aの時、Bを処理したい」「Aじゃないとき、Cを処理したい」といったことは、if文を使用することで実現できます。

例えば、「もしプレイヤーが空ならば、returnしたい」なら、次のようになります。


            auto player = Globals::ClientInstance->getLocalPlayer();
            if (!player) return;

ここのif (!player)は、if (player ==nullptr)と同じ意味の条件です。

上はreturn処理など、一つの処理のみを行っている場合ですが、複数の処理を行う時は{}（ブロック）を使う必要があります。 例えば、「もしプレイヤーが空ならば、bool変数dummyを偽にしてreturnしたい」という場合は、次のようになります。 


            auto player = Globals::ClientInstance->getLocalPlayer();
            if (!player) {
              dummy = false;
              return;
            }

では、そうでない場合はどう表現するのでしょうか。その場合は、else文を使用します。

if文と同様、複数の処理を行う場合はブロックを使用する必要があります。また、elseはifとくっ付けることができます。


            auto player = Globals::ClientInstance->getLocalPlayer();
            if (player) {
              dummy = true;
            }
            else {
              dummy = false;
              return;
            }
            

            int number = rand();
            if (number > 50) {
              std::cout << "A variable [number] is bigger than 50.\n";
            }
            else if (number < 50) {
              std::cout << "A variable [number] is smaller than 50.\n";
            }

コードを綺麗に書くために

C++に限らず、プログラミングではソースを閲覧する人にとって可読性が高く、安全なコードを意識することが大切です。以下に改善が見込めるコードの例を提示します。


            void findplayer() {
              if (ClientInstance::get()->getLocalPlayer() == nullptr) {
                return;
              }

              for (auto& actors : ClientInstance::get()->getLocalPlayer()->getLevel()->getRuntimeActorList()) {
                if (actors == nullptr || actors == ClientInstance::get()->getLocalPlayer()) {
                  continue;
                }
              }
            }

このコードの改善点を解説します。まず、関数名についてです。主にC++では、関数や変数に熟語の名前をつける際、一語目は小文字からはじめ、二語目以降は一文字目を大文字から始める傾向にあります。なので、ここでは関数名をfindPlayer()とすると良いです。

次に、条件文についてです。getLocalPlayer()など、何度も登場する関数は序盤で一つの変数に格納しておくべきです。例として以下のようになります。


            auto player = ClientInstance::get()->getLocalPlayer();

また、{}（ブロック）の使う場面についてですが、これに関してはTPOで考える必要があり、通常、条件文でブロックを用いるのは、複数の処理をある条件下で行うときです。今回の場合、2つのif文が登場していますが、どちらも一つの処理のみを行っているので、必ずではありませんが、ブロックは不要といえます。

加えて、プレイヤーが存在しない場合を条件とするには、player == nullptrよりも!playerの方が短く可読性が高いと考えられます。

以上を踏まえて修正したコードは以下のようになります。


            void findPlayer() {
              auto player = ClientInstance::get()->getLocalPlayer();
              if (!player) return;

              for (auto& actors : player->getLevel()->getRuntimeActorList()) {
                if (!actors || actors == player) continue;
              }
            }

これらの点に注意するだけで、今回の場合3行も削ることができるのです。不要なコードは後の開発やアプリの進行に大きく影響を与えるため、可読性と安全性を意識してコードを書くようにしましょう。
